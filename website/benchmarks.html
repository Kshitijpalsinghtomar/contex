Contex Benchmarks

Verified token reduction results from real test data across 21 dataset types, 5 output formats, and multiple row-counts.

Key Takeaways

  • 72% average token savings — measured as Contex Compact vs JSON across 21 dataset types at 1,000 rows (46–90% range).
  • 90% peak format savings — nested/repetitive JSON benefits most from template compression and dictionary encoding.
  • Zero data loss — 20/20 fidelity tests confirm lossless roundtrip across all formats and dataset types.
  • Sub-millisecond latency — encoding adds 3.8–79 μs/row depending on complexity; negligible vs API latency.
  • Deterministic output — same input always produces identical tokens, maximizing prefix cache hit rates.

When Contex Wins Most

Not all data compresses equally. Here's where Contex delivers the highest ROI.

Data Pattern Savings Tier Why
Nested / repetitive JSON 88–90% Template compression (@t) derives values from other columns
Sparse data (lots of nulls) 85–89% @sparse directive eliminates null columns entirely
Wide schemas (many columns) 79–84% Field name compression + constant column elision (@c)
API responses / healthcare 60–76% Dictionary encoding replaces repeated strings with @0, @1
Flat tabular data 58–78% Schema dedup + template + boolean/null abbreviation
Unique long-string data (e.g. GitHub API) 35–45% Format savings only — values are unique, less to compress

Methodology

Benchmarks are run using the v8 benchmark suite covering 21 dataset types.

Datasets (21 types)

Flat, RealWorld, NumericHeavy, Repetitive, Nested, DeepNested, MixedNested, Ecommerce, Healthcare, IoT, Sparse, ExtSparse, WideSchema, ChatMessages, LogEvents, Financial, GeoJSON, Multilingual, SocialGraph, TimeSeries, UserProfiles

Row Sizes

100 and 1000 rows per dataset type, plus 500 rows for pipeline/latency tests. Each combination run independently to measure scaling characteristics.

Output Formats

Contex Compact, TOON, CSV, Markdown, TENS-Text. All tested for both token count and data fidelity.

# Run the full benchmark suite
npx tsx packages/cli/src/benchmark.ts

# Analyze a specific dataset
contex analyze data.json --model gpt-4o-mini
